3.1. Getting started¶

Steps

Nisl provides a scikit-learn compliant transformer that helps loading Nifti files and provides some necessary preprocessing:

  1. Data loading : load Nifti files and check consistency of data
  2. Custom Masking : if a mask is not provided, computes one
  3. Preprocessings
  4. Temporal Filtering: detrending, confounds, normalization

3.1.1. Data loading¶

The typical way to load data is to use a fetching function provided by Nisl which will download a dataset and return a bunch of paths to the dataset files. Each returned object is dataset specific. In fact there is no generic dataset type in nisl: each dataset singularities are conserved. The functional files of the fetched dataset can then be passed directly to the NiftiMasker. For example :

from nisl import datasets
from nisl.io import NiftiMasker
dataset = datasets.fetch_nyu_rest(n_subjects=1)

We can now take a look at what Nisl loaded. It is only filenames referring to dataset files on the disk:

>>> dataset.keys()
['anat_skull', 'session', 'anat_anon', 'func']
>>> dataset.func 
['.../nyu_rest/session1/sub05676/func/lfo.nii.gz']
>>> dataset.session
[1]

We can also see that the data fetcher has generated a session array, this is explained in the advanced tutorial.

Note

Custom data

Please note that there are several ways to pass data to the Nifti Masker. You are not limited to the datasets proposed by nisl. Please take a look at Advanced Loading and Preprocessing of fMRI data to see how to use your own datasets or modify proposed dataset prior to passing them to the masker.

3.1.2. Masking (fit)¶

The main functionality of the Nifti Masker is obviously masking. It simply can apply a mask to your data, or generate one if you want. The great advantage of using the masker is that it can be easily embedded in a scikit-learn pipeline.

3.1.2.1. Mask Computing¶

If your dataset does not provide a mask, the Nifti masker will compute one for you. This is done in the fit step of the transformer. The generated mask can be accessed via the mask_ member and visualized.

nifti_masker = NiftiMasker(transpose=True)
nifti_masker.fit(dataset.func[0])
mask = nifti_masker.mask_img_.get_data()

Note

Transpose

You may notice that we initialized the masker with parameter transpose=True. This is because we will process our data with ICA that processes temporal series. Please see ICA of resting-state fMRI datasets page to understand what is done here. Understanding that is not required for this example.

3.1.2.2. Mask Visualization¶

We can easily get the mask from the Nifti Masker and display it. Here we use an EPI slice as a background and display the mask as a red overlay.

import pylab as pl
import numpy as np
import nibabel
pl.figure()
pl.axis('off')
pl.imshow(np.rot90(nibabel.load(dataset.func[0]).get_data()[..., 20, 0]),
          interpolation='nearest', cmap=pl.cm.gray)
ma = np.ma.masked_equal(mask, False)
pl.imshow(np.rot90(ma[..., 20]), interpolation='nearest', cmap=pl.cm.autumn,
          alpha=0.5)
pl.title("Mask")
pl.show()

3.1.3. Running default preprocessing (transform)¶

The transformer is a classic scikit-learn transformer. You can invoke him thanks to fit and transform methods or by a single call to fit_transform.

nifti_masker.fit(dataset.func[0])
fmri_masked = nifti_masker.transform(dataset.func[0])

3.1.4. Processing: running ICA¶

As a processing, we invoke scikit-learn ICA.

from sklearn.decomposition import FastICA
n_components = 20
ica = FastICA(n_components=n_components, random_state=42)
components_masked = ica.fit_transform(fmri_masked)

3.1.5. Unmasking (inverse_transform)¶

Unmasking data is as easy as masking it! Just call inverse_transform on your processed data.

components = nifti_masker.inverse_transform(components_masked)

3.1.6. Visualizing results¶

We visualize the result in a classical way. Here we show the default network. Please note that to end up with a better ICA map, some processing on the ICA maps is required: normalizing, centering and thresholding. Please see the ICA example for a more accurate representation of the maps.

components = np.ma.masked_equal(components.get_data(), 0)
pl.figure()
pl.axis('off')
pl.imshow(np.rot90(nibabel.load(dataset.func[0]).get_data()[..., 20, 0]),
          interpolation='nearest', cmap=pl.cm.gray)
pl.imshow(np.rot90(components[..., 20, 16]), interpolation='nearest',
          cmap=pl.cm.hot)
pl.show()

3.1.7. Using the power of pipeline¶

As said before, if the nifti masker is a scikit-learn transformer. As such, it can be integrated in a scikit-learn pipeline. The previous lines can be replaces by this more compact version:

from sklearn.pipeline import Pipeline
mask_ica = Pipeline([('masking', nifti_masker), ('ica', ica)])
components = nifti_masker.inverse_transform(
    mask_ica.fit_transform(dataset.func[0]))
